# SOW 生成器技能

根据**会议纪要 + 项目类型**，自动填充 SOW 模板，生成完整的项目工作说明书，并可选保存为飞书文档。

## 触发词

- "帮我生成 SOW"
- "根据会议纪要出工作说明书"
- "需求对接报告"
- "帮我 write SOW"
- "根据这个会议记录生成项目说明书"
- "生成项目工作说明书"
- "帮我出一份SOW"
- "会议纪要转SOW"

---

## 使用流程

### Step 1：收集信息

如果用户没有提供完整信息，按以下顺序引导补全：

#### 信息收集问卷

**【必填项】从会议纪要提取，提取不到则追问：**

| 序号 | 字段 | 说明 | 示例 |
|------|------|------|------|
| 1 | 客户名称 | 甲方公司全称 | 某某科技有限公司 |
| 2 | 项目名称 | 项目正式名称 | 智能客服系统建设项目 |
| 3 | 系统名称 | 拟建设的系统名称 | 某某智能客服平台 |
| 4 | 项目背景 | 客户痛点/现状描述 | 当前客服响应慢，人工成本高... |
| 5 | 核心诉求 | 一句话概括核心需求 | 建设AI智能客服系统，提升响应效率 |
| 6 | 核心功能需求 | 功能列表（至少3项） | 1.智能问答 2.工单管理 3.数据分析 |
| 7 | 终端类型 | PC端/小程序/H5/App | PC管理后台 + 微信小程序 |

**【选填项】提取不到则使用默认值或智能推断：**

| 序号 | 字段 | 默认值/推断规则 | 说明 |
|------|------|----------------|------|
| 8 | 项目类型 | 根据功能需求推断 | AI平台/Web系统/小程序/RAG/数据报表/混合 |
| 9 | 技术栈偏好 | 按项目类型推荐 | 见下方"项目类型技术推荐表" |
| 10 | 项目周期 | 根据功能复杂度推断 | 简单项目4-6周，复杂项目8-12周 |
| 11 | 乙方公司名称 | 从USER.md读取或询问 | 默认为"{{乙方公司名称}}" |
| 12 | 数据安全要求 | 无特殊要求 | 医疗/金融/儿童数据/跨境需特别标注 |
| 13 | 外部接口需求 | 无 | OCR/语音识别/第三方API等 |
| 14 | 二期功能 | 无 | 会议纪要中提到的后续功能 |
| 15 | 培训需求 | 2次免费培训 | 培训次数和超出费用 |

### Step 3：项目类型技术推荐

根据项目类型，自动推荐合适的技术方案：

| 项目类型 | 推荐技术栈 | 适用场景 |
|----------|-----------|----------|
| **AI + 数据平台** | FastAPI + MySQL + Claude/DeepSeek + AWS Bedrock | 智能分析、AI应用平台 |
| **纯 Web 系统** | Spring Boot / FastAPI + MySQL + React/Vue | 企业管理系统、OA、CRM |
| **小程序** | 微信小程序 + FastAPI + MySQL | 微信生态应用、轻量级工具 |
| **RAG 知识库** | FastAPI + OpenSearch/Milvus + 大模型 | 企业知识库、智能问答 |
| **数据报表系统** | FastAPI + MySQL/ClickHouse + ECharts | BI报表、数据可视化 |
| **混合（AI + 小程序）** | FastAPI + MySQL + 大模型 + 微信小程序 | AI能力+移动端应用 |
| **物联网平台** | FastAPI + MQTT + InfluxDB + MySQL | 设备管理、数据采集 |
| **电商系统** | Spring Boot / FastAPI + MySQL + Redis | B2B/B2C电商平台 |

#### 各项目类型默认配置

**AI + 数据平台：**
- 后端：FastAPI (Python)
- 数据库：MySQL 8.0
- AI模型：Claude 3.5 Sonnet / DeepSeek-V3
- 云服务：AWS Bedrock / 阿里云百炼
- 缓存：Redis
- 存储：AWS S3 / 阿里云OSS

**纯 Web 系统：**
- 后端：Spring Boot (Java) 或 FastAPI (Python)
- 数据库：MySQL 8.0 / PostgreSQL
- 前端：React 18 / Vue 3
- 云服务：阿里云 / 腾讯云
- 缓存：Redis
- 存储：OSS / MinIO

**小程序：**
- 前端：微信小程序原生 / Taro / UniApp
- 后端：FastAPI / Node.js
- 数据库：MySQL 8.0
- 云服务：腾讯云 / 阿里云
- 缓存：Redis
- 存储：COS / OSS

### Step 4：输出与保存

1. **生成预览**：将填充后的 SOW 以 Markdown 格式展示给用户
2. **标注待确认项**：所有 `[待确认]` 内容高亮显示，提醒用户核对
3. **询问保存方式**：
   - "SOW草稿已生成，是否需要保存到飞书文档？"
   - 如需要：调用 `feishu-doc-create-and-grant` 技能创建文档
   - 如不需要：提供 Markdown 文件下载

## 章节填充规则详解

### 从会议纪要提取的章节

| 章节 | 提取内容 | 处理方式 |
|------|----------|----------|
| 第一章 1.1 项目背景 | 客户现状、痛点 | 原文提取或概括 |
| 第一章 1.2 项目目标 | 功能目标列表 | 提取并格式化为列表 |
| 第一章 1.3 项目范围 | 功能模块 | 根据目标推断范围 |
| 第一章 1.4 交付内容 | 交付物 | 根据终端类型推断 |
| 第二章 技术方案 | 技术偏好 | 按项目类型推荐 |
| 第三章 界面功能 | 功能需求 | 按终端类型整理 |
| 第四章 外部接口 | 接口需求 | 提取第三方接口 |
| 第七章 7.1 甲方材料 | 需客户提供的内容 | 提取或推断 |
| 第八章 8.2 项目周期 | 时间要求 | 提取或按复杂度推断 |

### 直接复用的通用章节

以下章节内容完整，直接从模板读取，**不需要**从会议纪要提取：

- **第五章** 数据安全规划方案（5.1-5.4完整内容）
- **第六章** 变更流程管理（6.1-6.4完整流程）
- **第七章** 7.2 乙方职责（完整职责描述）
- **第八章** 8.1/8.3/8.4 项目管理（通用框架）
- **第九章** 项目约定（通用条款）
- **第十章** 项目培训（通用培训方案）
- **第十一章** 特别说明（通用声明）

## 注意事项

### 1. 数据安全补充规则

如项目涉及以下场景，在**第五章末尾**追加对应合规说明：

| 场景 | 补充说明 |
|------|----------|
| 医疗健康信息 | 需符合《个人信息保护法》《医疗健康数据安全指南》，涉及敏感个人信息需获得单独同意 |
| 金融数据 | 需符合《金融数据安全 数据安全分级指南》《个人金融信息保护技术规范》 |
| 儿童个人信息 | 需符合《儿童个人信息网络保护规定》，需监护人同意 |
| 跨境数据传输 | 需符合《数据出境安全评估办法》，通过安全评估或签订标准合同 |
| 人脸识别 | 需符合《人脸识别技术应用安全管理规定》，具有特定目的和充分必要性 |

### 2. 二期功能处理

如会议纪要中提到"后续"/"二期"功能：
- 在**第一章 1.3 项目范围**末尾添加："以下功能为二期规划，不在本期交付范围内：[功能列表]"
- 在**第十一章 特别说明**中添加二期功能规划说明
- 确保二期功能**不计入**项目周期和报价

### 3. 不确定内容标注

以下内容如无法从会议纪要明确提取，用 `[待确认]` 标注：
- 具体的功能细节描述
- 准确的项目周期（如客户有明确要求则使用）
- 外部接口的具体参数
- 数据安全特殊要求
- 培训次数和费用

**生成后必须提醒用户**："文档中标注 [待确认] 的内容需要您核对补充"

### 4. 版权声明

- 乙方公司名称优先从 `USER.md` 中读取 `company_name` 字段
- 如 USER.md 未配置，询问用户提供
- 如用户未提供，使用占位符 `{{乙方公司名称}}`

### 5. 默认值设置

| 字段 | 默认值 |
|------|--------|
| 免费培训次数 | 2次 |
| 超出培训费用 | 2000元/次 |
| 项目周期（简单） | 4-6周 |
| 项目周期（中等） | 6-8周 |
| 项目周期（复杂） | 8-12周 |
| 后端框架 | FastAPI |
| 数据库 | MySQL 8.0 |
| 云平台 | 阿里云 |

## 示例对话

**用户说：**
> 这是今天和客户开会的纪要，帮我生成一份项目工作说明书：
> 
> 会议时间：2024年3月15日
> 客户：ABC科技有限公司
> 参会人：张总（CEO）、李经理（IT负责人）
> 
> 会议内容：
> 1. ABC科技目前客服部门有20人，每天处理咨询量约500单，响应时间较长，客户满意度有待提升
> 2. 希望建设一套AI智能客服系统，实现自动回复常见问题
> 3. 需要PC端管理后台供客服人员使用，同时要有微信小程序供客户咨询
> 4. 核心功能：智能问答、工单管理、知识库管理、数据统计分析
> 5. 二期规划：接入语音机器人、多语言支持
> 6. 希望2个月内上线
> 7. 数据安全要求：客户咨询记录需要加密存储

**Agent 执行：**

1. 读取本 SKILL.md
2. 分析会议纪要，提取关键信息：
   - 客户名称：ABC科技有限公司
   - 项目类型：AI + 小程序（混合型）
   - 终端：PC端 + 微信小程序
   - 核心功能：智能问答、工单管理、知识库、数据分析
   - 二期功能：语音机器人、多语言支持
   - 周期：2个月（8周）
   - 数据安全：咨询记录加密
3. 读取 `templates/sow-template.md`
4. 按"AI + 小程序"类型填充技术栈：
   - 后端：FastAPI
   - 数据库：MySQL
   - AI：Claude/DeepSeek
   - 小程序：微信小程序
5. 填充所有占位符，二期功能单独标注
6. 展示预览，询问是否保存飞书文档
