## 1. 产品概述

### 1.1 背景

在 AI + 软件项目销售和交付过程中，需求对接完成后，项目经理需要手动整理会议纪要、撰写项目工作说明书（SOW）。这个过程耗时 2-4 小时，且格式不统一、易遗漏关键条款。

本技能基于真实项目 SOW（海底捞健康医疗体检报告智能系统）提炼通用模板，通过 AI 自动解析会议纪要、填充模板，将 SOW 生成时间压缩到 **5 分钟以内**。

### 1.2 产品定位

| 项目 | 说明 |
|------|------|
| 技能名称 | sow-generator |
| 技能类型 | OpenClaw Agent 技能（SKILL.md） |
| 运行环境 | OpenClaw 主 Agent（主会话直接调用） |
| 核心依赖 | SOW 模板文件 `templates/sow-template.md` |
| 可选依赖 | feishu-doc-create-and-grant 技能（保存飞书文档） |

### 1.3 核心价值

- **效率**：会议纪要 → 完整 SOW，5 分钟内完成
- **一致性**：所有 SOW 基于同一模板，格式统一
- **完整性**：通用法律条款（安全/变更/培训）自动填充，不遗漏
- **可扩展**：支持 8 种项目类型预设，覆盖主流 AI + 软件项目

## 2. 用户画像与使用场景

### 2.1 主要用户

**项目经理 / 售前顾问**
- 负责与客户对接需求
- 需要在会议后快速产出 SOW 文档
- 熟悉业务，但希望减少重复性文字工作

### 2.2 典型使用场景

**场景 A：会议结束后立即生成**
> 用户粘贴会议纪要文字，说"帮我生成 SOW"，Agent 直接输出完整草稿。

**场景 B：有部分信息，分步补全**
> 用户提供不完整的纪要，Agent 识别缺失信息后追问，用户补充后生成。

**场景 C：生成后保存飞书文档**
> 用户确认草稿后，Agent 自动创建飞书文档并授权给用户查看。

**场景 D：基于已有纪要修改**
> 用户说"把客户名改成 XX，周期改成 3 个月"，Agent 更新对应字段后重新输出。

## 3. 功能需求

### 3.1 核心功能（必须实现）

#### F1：触发识别

| 需求 ID | F1 |
|---------|-----|
| 功能名称 | 触发词识别 |
| 描述 | Agent 能识别用户意图为"生成SOW"并加载本技能 |
| 触发词列表 | "帮我生成 SOW"、"根据会议纪要出工作说明书"、"需求对接报告"、"帮我写 SOW"、"根据这个会议记录生成项目说明书"、"生成项目工作说明书"、"帮我出一份SOW"、"会议纪要转SOW" |
| 优先级 | P0 |

---

#### F2：信息提取

| 需求 ID | F2 |
|---------|-----|
| 功能名称 | 从会议纪要提取关键信息 |
| 描述 | Agent 分析会议纪要文本，自动提取填充 SOW 所需的关键字段 |
| 优先级 | P0 |

**提取字段清单：**

| 字段 | 类型 | 提取规则 | 不可提取时处理 |
|------|------|----------|---------------|
| 客户名称 | 必填 | 从纪要"客户/甲方/公司名"提取 | 追问用户 |
| 项目名称 | 必填 | 从纪要提取或推断（"XX系统建设项目"） | 追问用户 |
| 系统名称 | 必填 | 从纪要提取或推断（"XX智能平台"） | 基于功能推断 |
| 项目背景 | 必填 | 提取客户痛点和现状描述段落 | 追问用户 |
| 核心诉求 | 必填 | 一句话概括（提取或 AI 总结） | AI 基于纪要生成 |
| 核心功能列表 | 必填 | 提取功能需求，格式化为列表 | 追问用户 |
| 终端类型 | 必填 | 识别 PC/小程序/H5/App/Web | 追问用户 |
| 技术偏好 | 选填 | 提取客户提到的技术/云平台偏好 | 按项目类型推荐 |
| 项目周期 | 选填 | 提取时间要求（"X个月内上线"） | 按复杂度推断 |
| 二期功能 | 选填 | 提取"后续/二期/下阶段"相关内容 | 留空 |
| 外部接口 | 选填 | 提取第三方 API/接口需求 | 留空 |
| 数据安全特殊要求 | 选填 | 识别医疗/金融/跨境等关键词 | 无补充说明 |

---

#### F3：模板填充

| 需求 ID | F3 |
|---------|-----|
| 功能名称 | SOW 模板填充 |
| 描述 | 将提取的信息填入 `templates/sow-template.md` 所有 `{{占位符}}` |
| 优先级 | P0 |

**填充规则：**

1. 所有 `{{占位符}}` 必须被替换，不允许留空占位符在最终输出中
2. 无法确定的内容用 `[待确认]` 替换，不能留 `{{占位符}}`
3. 通用章节（第五、六、八、九、十、十一章）直接保留完整内容，不简化
4. 二期功能在第一章 1.3 末尾和第十一章中单独标注

---

#### F4：项目类型技术推荐

| 需求 ID | F4 |
|---------|-----|
| 功能名称 | 按项目类型自动推荐技术栈 |
| 描述 | 根据识别的项目类型，自动填充技术方案章节 |
| 优先级 | P0 |

**项目类型判断规则：**

| 判断依据 | 项目类型 |
|----------|----------|
| 有 AI/大模型需求 + 无移动端 | AI + 数据平台 |
| 有 AI/大模型需求 + 有小程序 | 混合（AI + 小程序） |
| 有小程序需求 + 无 AI 需求 | 小程序 |
| 有向量数据库/知识库/RAG 需求 | RAG 知识库 |
| 有报表/图表/BI 需求 + 无 AI | 数据报表系统 |
| 有传感器/IoT/设备数据需求 | 物联网平台 |
| 纯企业管理系统 | 纯 Web 系统 |

**各类型默认技术栈：**

| 项目类型 | 后端 | 数据库 | AI | 云 | 存储 |
|----------|------|--------|----|----|------|
| AI + 数据平台 | FastAPI | MySQL 8.0 | Claude 3.5 Sonnet | AWS Bedrock | S3 |
| 混合（AI + 小程序） | FastAPI | MySQL 8.0 | DeepSeek-V3 / Claude | 阿里云 | OSS |
| 小程序 | FastAPI / Node.js | MySQL 8.0 | - | 腾讯云 | COS |
| RAG 知识库 | FastAPI | MySQL + Milvus | 大模型（通用） | 阿里云 | OSS |
| 数据报表系统 | FastAPI | ClickHouse / MySQL | - | 阿里云 | OSS |
| 物联网平台 | FastAPI | InfluxDB + MySQL | - | 阿里云 / AWS | OSS |
| 纯 Web 系统 | Spring Boot | MySQL 8.0 | - | 阿里云 | OSS |

---

#### F5：SOW 输出

| 需求 ID | F5 |
|---------|-----|
| 功能名称 | 输出完整 SOW |
| 描述 | 以 Markdown 格式输出完整的 11 章节 SOW，包含所有通用条款 |
| 优先级 | P0 |

**输出要求：**
- 格式：Markdown
- 章节：完整 11 章，无遗漏
- 标注：所有 `[待确认]` 内容在输出后提醒用户核对
- 长度：通常 3000-6000 字

#### F6：追问补全

| 需求 ID | F6 |

#### F7：保存飞书文档

| 需求 ID | F7 |
|---------|-----|
| 功能名称 | 保存为飞书云文档 |

| 优先级 | P1 |
| 前提条件 | feishu-doc-create-and-grant 技能已安装 |

#### F8：数据安全合规补充

| 需求 ID | F8 |
|---------|-----|
| 功能名称 | 特殊数据安全合规说明自动补充 |
| 描述 | 识别医疗/金融/儿童/跨境等敏感场景，在第五章自动追加对应合规说明 |
| 优先级 | P1 |

**触发关键词：**

| 关键词 | 追加内容 |
|--------|----------|
| 医疗/体检/病历/健康数据 | 《个人信息保护法》医疗健康敏感信息合规说明 |
| 金融/银行/支付/账户 | 《金融数据安全》合规说明 |
| 儿童/未成年/学生 | 《儿童个人信息保护》合规说明 |
| 跨境/海外/出境 | 《数据出境安全评估》合规说明 |

---

### 3.3 不在范围内（明确排除）

- ❌ 自动生成报价/预算表（SOW 不含价格，价格单独签合同）
- ❌ 自动发送文档给客户（需用户手动操作）
- ❌ 多语言 SOW（英文版为后续规划）
- ❌ 自动从飞书会议纪要同步（需用户手动粘贴）

## 4. 数据流与处理逻辑

```
用户输入（会议纪要文字）
        │
        ▼
[F1] 触发识别：是否为 SOW 生成意图？
        │ Yes
        ▼
[F2] 信息提取：解析纪要，提取关键字段
        │
        ├─── 缺失必填字段？ ──Yes──→ [F6] 追问用户 ──→ 补全字段
        │ No
        ▼
[F4] 项目类型判断 → 推荐技术栈
        │
        ▼
[F3] 模板填充：读取 sow-template.md，替换占位符
        │
        ├─── 涉及敏感数据？ ──Yes──→ [F8] 追加合规说明
        │
        ▼
[F5] 输出 SOW（Markdown）
        │
        ├─── 有 [待确认]？ ──Yes──→ 提醒用户核对
        │
        ▼
询问：是否保存飞书文档？
        │ Yes
        ▼
[F7] 调用 feishu-doc-create-and-grant 技能
```

## 5. 输入/输出规范

### 5.1 输入规范

| 输入类型 | 支持 | 说明 |
|----------|------|------|
| 纯文字会议纪要 | ✅ | 最主要的输入方式 |
| 结构化纪要（含表格） | ✅ | AI 可解析 Markdown/HTML 表格 |
| 口述补充 | ✅ | 用户在追问时口头补充 |
| 飞书文档链接 | ⚠️ 部分支持 | 需先读取文档内容再处理 |
| 录音文字转写 | ✅ | 同纯文字处理 |

### 5.2 输出规范

**SOW 文档结构：**

```
# [客户名称] [项目名称]
## 项目工作说明书（SOW）

版权声明
内容修订记录
目录

第一章  项目概述（背景/目标/范围/交付物）
第二章  技术方案（架构/技术栈/数据库/云服务）
第三章  界面功能设计（按终端分节）
第四章  外部接口需求
第五章  数据安全规划方案（通用 + 场景补充）
第六章  变更流程管理（通用）
第七章  项目职责分工（甲方材料 + 乙方职责）
第八章  项目实施和管理（通用）
第九章  项目约定（通用）
第十章  项目培训（通用）
第十一章 特别说明（通用 + 二期规划）
```

**质量标准：**

| 指标 | 标准 |
|------|------|
| 章节完整性 | 11 章节全部存在，无遗漏 |
| 占位符清零 | 输出中不含任何 `{{xxx}}` 格式占位符 |
| 待确认标注 | 不确定内容使用 `[待确认]`，不留空白 |
| 通用章节 | 第五/六/八/九/十/十一章内容完整，不被简化 |
| 文档长度 | 不低于 3000 字（中文） |

## 6. 技术实现规范

### 6.1 文件结构

```
skills/sow-generator/
├── SKILL.md                    # 技能主文件（触发词、流程、规则）
├── PRD.md                      # 本文档（产品需求文档）
└── templates/
    └── sow-template.md         # 通用 SOW 模板（带占位符）
```

### 6.2 技能加载规则

- Agent 在 `available_skills` 中扫描到本技能的 `description` 匹配时，读取 `SKILL.md`

## 8. 验收标准

### 8.1 功能验收（每个测试用例都需通过）

| 验收项 | 检查方式 | 通过标准 |
|--------|----------|----------|
| 触发识别 | 说"帮我生成SOW" | Agent 加载本技能，不走其他流程 |
| 章节完整 | 检查输出目录 | 11个章节全部存在 |
| 占位符清零 | 搜索 `{{` | 输出中不含任何 `{{` |
| 信息提取准确 | 对比纪要和输出 | 客户名、功能列表、终端类型与纪要一致 |
| 技术栈推荐 | 检查第二章 | 技术栈与项目类型匹配（见第 4 节推荐表）|
| 通用章节保留 | 检查5/6/8/9/10/11章 | 内容完整，与模板基本一致，未被简化 |
| 待确认标注 | 搜索 `[待确认]` | 不确定内容有标注，且输出后有提醒 |
| 文档长度 | 字数统计 | 不低于 3000 中文字 |

### 8.2 医疗场景专项验收（TC-01）

| 验收项 | 通过标准 |
|--------|----------|
| 合规说明 | 第五章包含医疗健康数据合规说明 |
| AI 技术栈 | 第二章包含大模型相关技术（AWS Bedrock / 阿里云百炼等）|
| 数据安全 | 提及体检数据加密存储、访问控制 |

### 8.3 物联网场景专项验收（TC-04）

| 验收项 | 通过标准 |
|--------|----------|
| 技术栈 | 第二章包含 InfluxDB 或时序数据库 |
| 接口需求 | 第四章包含传感器数据采集相关接口 |

### 8.4 性能验收

| 指标 | 标准 |
|------|------|
| 响应时间 | 从输入到输出完整 SOW，不超过 120 秒 |
| 追问次数 | 在纪要信息完整时，不追问；信息缺失时追问不超过 1 轮（3 个问题） |
